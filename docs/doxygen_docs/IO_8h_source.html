147 int value = 256 487 423
148 Sa representation binary avec les bits de poids fort a gauche est :
149 0000 1111 0100 1001 1010 1111 1111 1111
150 En hexadecimal cela devient:

152 Cet entier peut être stocke dans un tableau de 4 caracteres de 1 octet chacun par:
153 char *tableau = reinterpret_cast<char *>(value)
154 Chaque octet correspond alors a un caractere de la table ASCII (sur 8 bits)

156 ----------BASE64-------------
157 
158 Base64 : L'objectif est d'encoder chaque groupe de 24 bits successif par une chaîne de 4 caracteres simples.
159 Pour passer en Base64 soit sur des multiples de 24 bits (4 x 6bits), on decoupe
160 la representation binary en paquet de 6 bits. Dans notre cas, cela devient:
161 000011 110100 100110 101111 111111 11
162 le dernier etant incomplet on complete avec des 0 :
163 000011 110100 100110 101111 111111 110000
164 
165 On associe ensuite a chaque code binary sur 6 bits un caractere simple parmis les 64 suivants:
166 "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/"
167 
168 Ainsi, la transcription en Base64 donnera:
169  D 0 m v / w
170 
171 Tout ceci est stocke a nouveau dans des chaînes de caractere (8 bits par caractere),
172 on complete alors par deux bits nuls devant chaque jeu de 6 bits :
173 00000011 00110100 00100110 00101111 00111111 00110000
174 on complete egalement pour obtenir un multiple de 4 caracteres avec le caractere '=':
175 
176 Ainsi la transcription final de notre entier en Base64 sera :
177  D0mv/w==
178 
179 qui aura la representation binary final (Pas sur pour le = si c'est sa representation ASCII ou pas)
180 00000011 00110100 00100110 00101111 00111111 00110000 00111101 00111101
